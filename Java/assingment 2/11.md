# Q11 – Demonstrate `this` Keyword in Java

### Concept

The **`this` keyword** refers to the **current object of the class**.

It is commonly used when **instance variables and parameters have the same name**.

---

# 💻 Java Program

```java
class Student {

    int id;
    String name;

    Student(int id, String name) {

        this.id = id;
        this.name = name;
    }

    void display() {
        System.out.println("ID: " + id);
        System.out.println("Name: " + name);
    }

}

public class ThisDemo {

    public static void main(String args[]) {

        Student s1 = new Student(101, "Rahul");
        Student s2 = new Student(102, "Amit");

        s1.display();
        s2.display();
    }
}
```

---

# Program Flow

### Step 1 – Object Creation

```
Student s1 = new Student(101,"Rahul");
```

Constructor receives

```
id = 101
name = Rahul
```

---

### Step 2 – `this` Keyword

Inside constructor:

```
this.id = id
```

Meaning

```
object variable = parameter value
```

So

```
s1.id = 101
s1.name = Rahul
```

---

### Step 3 – Second Object

```
Student s2 = new Student(102,"Amit")
```

Stored as

```
s2.id = 102
s2.name = Amit
```

---

# Output

```
ID: 101
Name: Rahul
ID: 102
Name: Amit
```

---

# Why `this` is Needed

Without `this`:

```
id = id
```

Java gets confused because both names are same.

With `this`:

```
this.id = id
```

Now Java understands:

```
this.id → object variable
id → constructor parameter
```

---

# Visual Idea

```
Object s1

s1
 ├─ id = 101
 └─ name = Rahul
```

`this` refers to **that object**.

---

# Common Beginner Mistake

Wrong

```
id = id
name = name
```

Correct

```
this.id = id
this.name = name
```

---

# Where `this` is Used

1️⃣ Inside constructors
2️⃣ Calling current class methods
3️⃣ Passing current object as parameter
